Standard quality process for ALL external actions:

1. **Do the work**
2. **Verify it actually worked** — don't assume success from a 200 status or "clicked" log. Navigate to the result and confirm visually.
3. **Retry if failed** — if verification shows it didn't work, try again before reporting done.
4. **Screenshot each completed step** — save to /tmp/ with descriptive names.
5. **Send a verification report** — to Telegram or the user with:
   - What was done
   - Screenshots proving it's done
   - Any items that failed or need manual followup
   - Links to verify (public URLs where applicable)

**Why:** User feedback 2026-04-10: "always do double check on everything you done, and try again if it failed, take screenshots of each page that is done and send a report to me if you verify its actually done"

**How to apply:** Every directory submission, social media post, invoice publish, code deploy, profile update — verify the end result exists and is correct before reporting success.
